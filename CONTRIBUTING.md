## How to Contribute (For Future Reference)

When external contributions are enabled, we will welcome contributions to the project! If you have an idea for a new feature, bug fix, or improvement, please follow
these steps:

1. Check if there is already an issue for your idea in
   the [issue tracker](https://github.com/AmadeusITGroup/Dataspace_Ecosystem/issues). If there is already an issue, feel
   free to comment on it with your thoughts or suggestions. If not, open
   a [discussion](https://github.com/AmadeusITGroup/Dataspace_Ecosystem/discussions) to propose your idea and wait
   until this discussion is converted into an issue.
2. Fork the repository and clone it to your local machine.
3. Create a new branch for your changes. Use a descriptive name for the branch that reflects the purpose of your
   changes (e.g., `feature/new-feature-name` or `bugfix/fix-issue-name`).
4. Make your changes and commit them with a clear message to explain what you have done. Please follow the
   [Dataspace Ecosystem Code Style Guide](STYLE_GUIDE.md) for code formatting and style.
5. Push your changes to your forked repository.
6. Create a pull request to the main branch of the repository. In your pull request, please provide a clear
   description of your changes and why they are necessary following the [PR Etiquette](PR_ETIQUETTE.md)
7. Wait for the review process. The maintainers will review your changes and provide feedback. Be open to
   suggestions and be ready to make any necessary adjustments.
8. Once your changes are approved, they will be merged into the master branch of the repository.
9. Congratulations! You have successfully contributed to the Dataspace Ecosystem project.

If you have any questions or need assistance, feel free to reach out to the maintainers or the community for help.
